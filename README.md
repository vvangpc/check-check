# Check-Check 专利形式审查桌面端

Check-Check 是一套专为中国专利实务打造的、高性能且高度现代化的本地专利文本形式审查桌面工具。它能够有效识别专利申请文件中的“逻辑缺陷”、“附图标记冲突”以及“敏感词违规”。

## 🌟 核心特性

- **多线程/高性能架构**：应用采用 `PyQt6` 以及异步工作线程完成大规模文本正则检索，不论是十几页还是五十页的长文均能快速通过审查而不卡死。
- **动态无损高亮引擎**：不仅有基于 `QSyntaxHighlighter` 机制产生的实时敏感词底色警告，针对跨段落上下文运算发现的逻辑断裂错误，系统也会跨文本精准染色，同时支持双击自动滚动跳转。
- **一键智能分词/分割**：自动读取 `.docx`, `.rtf`, `.txt` 等多种格式专利底稿，并运用 NLP 特征将“权利要求书”与“说明书”准确剥离并放入比对框内。
- **主题自由定制**：内置了包括暗色、亮色工作流引擎的主题定制系统。支持对说明书、权要文本框以及审查报告分别进行护眼色的定制。

## 🔍 重磅审查逻辑矩阵

这套软件并非只做基础提取，而是对专利常犯的隐性“致命缺陷”做了严谨的技术防备：

1. **附图标记智能校验 (防交叉粘连错乱)**
   - 包含未完全闭合括号检测（例如 “夹指2)” 或 “夹指(2”）机制。
   - **异名同号 / 同名异号深层检测**：基于内置的“最长公共后缀聚类算法”，能够彻底规避常由中文语法粘连导致的误报（如“由于夹指”与“驱动夹指”可自动消抖合并为核心词“夹指”），不再乱弹错。
2. **缺乏前序基础校验 (滑单步匹配前向扫描)**
   - 使用了长度渐减的溯源防误伤逻辑。若发现特定冠词“所述”、“该”紧跟新部件，系统会在它的前置推导长廊里利用前缀滑动检索；能够完美豁免长修饰短语引发的伪缺乏错误，并自动过滤“发明”、“系统”、“步骤”等通用词汇。
3. **依赖权项多段落引用的合法性检验**
   - 多引多排雷：如果多项从属权利要求引用了具有“多引用特征”的源头，抛出致命警报。
   - 引用断层排雷：判断引用自己或后文的不合规“跳项”。
   - 环状引证（循环依赖）排雷：递归校验出死循环的逻辑网络。
4. **引用主题一致性拦截**
   - 当从权要求“根据权利要求X所述的Y”时，它必须和权利要求X本身的特征主体描述 Y（或“一种Y”）保持完全吻合，防范撰写不规范。
5. **缺少说明书支持验证库 (支持同义词加载)**
   - 支持动态在 `dicts/synonyms.txt` 添加你专属行话的同义词集。自动核对权利要求内的实词是否真的被涵盖在了说明书主文中发挥技术支撑作用。

## 🚀 启动指引

请确保您的计算机上已安装 Python 环境（项目推荐使用 `uv`，也支持常规 `pip`）。

```bash
# 1. 下载至本地
git clone https://github.com/vvangpc/check-check.git
cd check-check

# 2. 安装项目依赖 (如果有)
# uv pip install -r requirements.txt
# 或者手动安装依赖
pip install PyQt6 python-docx

# 3. 运行审查工具的界面！
python main.py
```

## 📝 基本操作与提示

1. 点击 **"上传文件及自动分割"** 导入撰写好的专利文件。
2. 切割完成后，您可以任意检视左右双分栏里的权利要求与说明书。
3. 如果发觉有少许偏差，直接原地输入/修改切割即可，不会影响底稿。
4. 点击 **"搜索"**：焦点位于哪个文框（说明书/权要），搜索框结果就会自动对它执行查找追踪。
5. 点击顶部的 **"开始形式审查"**！耐心等待一至两秒。
6. 右下方的智能报告单出具后，请直接**双击警告项目**，光标将自动把您带往案发现场（带标红高亮）。
